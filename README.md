***********************************************
Post Office Chat-room
***********************************************

Программа "Post Office Chat-room" - десктопное приложение реализующее сетевой чат.

Основные моменты:
•	Клиент-серверное приложение состоящее из двух модулей (server, client) взаимодействующих между собой посредством интерфейса сокетов.
•	При построении проекта использовался Apache Maven
•	Пользовательский интерфейс написан на JavaFX с использованием  CSS
•	Для хранения и создания учётных данных пользователей используется БД MySQL 

Реализовано:
•	хранение истории сообщений в БД MySQL при отправке служебного сообщения "/history"
•	занесение произвольного пользователя в "черный список" при отправке служебного сообщения "/blacklist"
•	отправка персональных (приватных) сообщений
•	логирование основных событий

Monster Hunter
***********************************************

Программа "Monster Hunter" - десктопная Action/RPG игра написанная на фреймворке LibGDX

Сценарий: Герой бродит по лесу и уничтожает враждебных тварей. Получает за это вознаграждение и лечение, подбирает оружие. 

Управление героем производится с клавиатуры. Клавиши A,W,D,S отвечают за направление движения. SHIFT - ускорение движения. P,L - атака (меч и стрелы). O - перебор инвентаря. SPACE - применение выбранного элемента инвентаря.



***********************************************
Post Office-Cloud storage
***********************************************

Программа "Post Office-Cloud storage" - десктопное приложение реализующее сетевой хранилище.

Основные моменты:
•	Клиент-серверное приложение состоящее из трёх модулей (server, client, common) взаимодействие между сервером и клиентом организовано посредством интерфейса сокетов.
•	При построении проекта использовался Apache Maven
•	Пользовательский интерфейс написан на JavaFX с использованием CSS
•	Для хранения и создания учётных данных пользователей используется БД MySQL 

Реализовано:
•	создание новых пользователей и хранение их данных в БД MySQL. Также при создании нового пользователя на стороне сервера 		создаётся именная папка для хранения файлов данного пользователя. Каждый пользователь имеет доступ только к своей папке.
•	передача файлов между клиентом и сервером производится в виде сериализованных объектов, при этом исходный файл передается 	частями по 10 Мбайт
•	в пользовательском интерфейсе отображается актуальный список файлов находящийся в хранилище, как на стороне клиента, так и 		на стороне сервера.
•	логирование основных событий
